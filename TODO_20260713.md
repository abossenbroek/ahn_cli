## Two hard realities (do not relitigate)

- **We CAN produce a gaussian-splat *encoding* of our data; we CANNOT produce a *trained
  radiance field*.** Proven by the spike (`scratchpad/bevy-splat-spike/src/amsterdam.rs`):
  our `tiles.hfp` heightfield pack → one isotropic gaussian per cell (ENU pos + NAP height,
  ortho colour as SH deg-0), 265k gaussians @ ~60 FPS in `bevy_gaussian_splatting`. That is
  a real, deterministic 3DGS cloud. RadSplat/SMERF (view-dependent, multi-view-trained) stay
  blocked — no multi-view capture. Track B ships the geometric encoding, honestly labelled.
- **Draco: rejected, no measurement.** Geometry is already `KHR_mesh_quantization` 16-bit +
  `EXT_meshopt_compression` on a regular grid; `.hf` collapses it to 12-bit implicit-XY.
  Draco re-compresses already-twice-reduced geometry at ~10–50× decode cost for ~zero gain.
  The texture is the byte lever → **KTX2/Basis** (GPU-native, VRAM, transcode-at-load).

## Guardrails (every item)

- **DDD.** `tiles3d` stays one bounded context; the encoder seam is `TilePayload` → `TileEncoder`.
  New profiles plug into `profile.py`/`encoders.py`; only encoder-layer modules know codec bytes.
- **Determinism.** Every producer output byte-frozen; `verify.py` re-reads from disk and byte-
  compares against an independent rebuild. New codecs must be deterministic on a given machine
  with a pinned lib version (the JPEG contract).
- **100% branch coverage**, pyright strict, ruff ALL, `make check` green (Python tracks A/B).
- **Rust:** `make rust-test` green; `ahn-heightfield` MSRV 1.77 / cargo-deny untouched.

## Track B — `splat` producer profile (`content_kind = 2`) — ✅ COMPLETE (51e4c52..124285a, review clean)

New profile `splat`: encode each tile's heightfield+ortho as a gaussian-splat blob (spike
algorithm in the deterministic producer). One gaussian per cell at the LOD stride: ENU pos +
NAP height, ortho colour SH deg-0, isotropic scale ≈ cell·stride, identity rotation, opacity 1.
Texture **forbidden** (colour in SH) → fits the two-blob entry, no format-version bump.

Blob codec: deterministic little-endian **binary 3DGS `.ply`** (`x y z, scale_0..2, rot_0..3,
opacity, f_dc_0..2`) **zstd-wrapped** (mirrors `.hf` framing; `bevy_gaussian_splatting` `io_ply`
reads it). Size caveat: ~66k gaussians × ~56 B ≈ 3.7 MB raw/tile before zstd — bound via stride
+ optional region-relative position quantization. Document it.
