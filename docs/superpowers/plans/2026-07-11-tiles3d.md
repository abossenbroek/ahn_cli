# tiles3d (OGC 3D Tiles 1.1) + hard dimension gates — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a `tiles3d` bounded context converting ortho.tif + reconcile's
EXR into an OGC 3D Tiles 1.1 tileset with a strictest-possible post-write
verifier, and tighten reconcile so partial coverage / void estimates are
hard errors.

**Architecture:** New `ahn_cli/tiles3d/` package mirroring the `copc/`
pattern (typed `Tiles3dError`, `build_tiles3d()` orchestrator, CLI adapter
in `cli/app.py`). Quadtree of stride-sampled heightfield meshes; every
vertex/texel is a genuine source sample. All writers hand-packed and
byte-deterministic. The verifier re-reads every artifact from disk,
validates it structurally against the spec, and compares it **byte-exact**
against an independent rebuild from the sources.

**Tech Stack:** numpy, rasterio (ortho read), pyproj (EPSG:7415→4978/4979,
NEW direct dep), stdlib zlib/struct/json (PNG, glb, tileset.json).
Spec: docs/superpowers/specs/2026-07-11-tiles3d-design.md.

## Global Constraints

- 100% branch coverage on `ahn_cli/` non-legacy code (`fail_under = 100`).
- pyright strict; ruff `select=["ALL"]`, line length 78; `make check` green.
- Byte-determinism: no timestamps, sorted JSON keys, pinned zlib level.
- Data is never infilled: every mesh vertex / texel is a genuine sample.
- Bounded contexts: tiles3d imports only `domain/` from other contexts
  (tests may import `reconcile` writers to build fixtures).
- Every failure is `Tiles3dError` (or `ReconcileError` in reconcile);
  the CLI translates to `click.ClickException`.
- Failed builds leave **nothing** behind (unlink outputs on any error).

### Task 1: Reconcile coverage gate (bbox containment, not overlap)

**Files:**
- Modify: `ahn_cli/reconcile/reconcile.py` (`_verify_cloud_overlaps_grid` → `_verify_cloud_covers_grid`, call site line 168, docstring failure-modes)
- Test: `tests/reconcile/test_reconcile.py` (or wherever overlap tests live — find with `grep -rn "does not overlap" tests/`)

**Interfaces:**
- Produces: `_verify_cloud_covers_grid(coords, grid, request) -> None`,
  raising `ReconcileError` unless the cloud XY bbox contains the full
  pixel-CENTRE extent (cols/rows at `0.5 … dim-0.5` through the affine
  transform, all four corner combinations).

Gate logic (replaces the `disjoint` test):

```python
t = grid.transform
cols = np.array([0.5, grid.width - 0.5])
rows = np.array([0.5, grid.height - 0.5])
xs = t[0] * cols[:, np.newaxis] + t[1] * rows[np.newaxis, :] + t[2]
ys = t[3] * cols[:, np.newaxis] + t[4] * rows[np.newaxis, :] + t[5]
shortfalls = [
    ("west", float(xs.min()) - float(coords[:, 0].min())),
    ("east", float(coords[:, 0].max()) - float(xs.max())),
    ("south", float(ys.min()) - float(coords[:, 1].min())),
    ("north", float(coords[:, 1].max()) - float(ys.max())),
]
uncovered = [(side, -gap) for side, gap in shortfalls if gap < 0]
if uncovered: raise ReconcileError(<message naming each side and metres>)
```

- [ ] Failing tests: cloud shifted by half the grid extent (partial
  overlap) → `ReconcileError` matching `"does not cover"` and the side
  name; cloud exactly covering the pixel-centre extent → passes. Keep the
  fully-disjoint test passing (update its expected message).
- [ ] Implement; run `uv run pytest tests/reconcile -x -q`.
- [ ] `make check`; commit `feat(reconcile): require full cloud coverage of the ortho grid`.

### Task 4: strict EXR reader (`tiles3d/exr.py`)

**Files:**
- Create: `ahn_cli/tiles3d/exr.py`
- Test: `tests/tiles3d/test_exr.py`, `tests/tiles3d/conftest.py`

**Interfaces:**
- Produces:
  ```python
  @dataclass(frozen=True)
  class ReconciledExr:
      width: int; height: int
      x: npt.NDArray[np.float32]  # (h, w); likewise y, z, r, g, b

  def read_reconciled_exr(path: Path) -> ReconciledExr  # raises Tiles3dError
  ```
- Consumes: byte layout of `ahn_cli/reconcile/writers.py::_ExrWriter` /
  `_exr_header` (magic `0x01312F76`, version 2, attributes in written
  order: channels chlist [B,G,R,X,Y,Z FLOAT, pLinear 0, sampling 1,1],
  compression 0, dataWindow/displayWindow box2i `(0,0,w-1,h-1)`, lineOrder
  0, pixelAspectRatio 1.0, screenWindowCenter (0,0), screenWindowWidth
  1.0, `\x00` terminator; then `h` uint64 scanline offsets that must equal
  `header_len + h*8 + row*(8 + w*4*6)`; each scanline `int32 y == row`,
  `int32 size == w*4*6`, then 6 channel-planar float32 rows in B,G,R,X,Y,Z
  order; EOF exactly after the last scanline).

Every deviation → `Tiles3dError` with a distinct message (bad magic /
version / unknown or missing attribute / wrong channel set / compression /
nonzero data-window origin / display≠data window / bad offset table /
scanline y or size mismatch / truncated / trailing bytes).

- [ ] `tests/tiles3d/conftest.py`: `make_exr(tmp_path, grid, mask=None)`
  factory using `ahn_cli.reconcile.writers.write_reconciled(OutputFormat.EXR, …)`
  (test-only cross-import) and a `corrupt(path, offset, new_bytes)` helper.
- [ ] Failing tests: round-trip equality (x/y/z/r/g/b bit-exact vs the
  arrays fed to the writer, incl. `r == float32(rgb/255.0)`), then one
  negative test per deviation via byte surgery.

### Task 5: sources + the perfect-dimension-match gates (`tiles3d/sources.py`)

**Files:**
- Create: `ahn_cli/tiles3d/sources.py`
- Test: `tests/tiles3d/test_sources.py` (+ conftest: `make_ortho(tmp_path,
  rgb, transform)` writing an EPSG:28992 uint8 GeoTIFF via rasterio, like
  `tests/reconcile/conftest.py`)

**Interfaces:**
- Produces:
  ```python
  @dataclass(frozen=True)
  class TerrainGrid:
      width: int; height: int
      transform: tuple[float, float, float, float, float, float]
      x: npt.NDArray[np.float32]; y: ...; z: ...   # (h, w)
      rgb: npt.NDArray[np.uint8]                   # (h, w, 3)

  def load_terrain(ortho_path: Path, heights_path: Path) -> TerrainGrid
  ```
- Consumes: `read_reconciled_exr`, `ahn_cli.domain.authenticity`
  (`uniform_image`, `flat_surface`).

Gates, in order, each `Tiles3dError`:
1. ortho unreadable / `count < 3` / dtype not uint8 / CRS not EPSG:28992.
2. `uniform_image` on a ≤512² decimated 3-band sample (placeholder guard).
3. EXR gates (Task 4) via `read_reconciled_exr`.
4. **dimensions equal**: EXR w/h == ortho w/h.
5. **X/Y planes bit-exact** vs `float32` of pixel centres from the ortho
   transform, computed with the same arithmetic as
   `reconcile/raster.py::block_target_coordinates`
   (`x = t0*(col+0.5) + t1*(row+0.5) + t2`, float64 then `astype(np.float32)`);
   first mismatching pixel named in the error.
6. **RGB bit-exact**: `exr.r == (ortho_band0/255.0).astype(np.float32)`
   etc. — proves the EXR was reconciled from THIS ortho.
7. **nothing missing**: `np.isfinite(z).all()` else error naming count;
   `flat_surface(z, None)` → error (placeholder surface).

- [ ] Failing tests: happy path (build EXR from the ortho's own pixel
  centres/colours); each gate negative (2-band tiff, float tiff, wrong
  CRS, uniform ortho, mismatched size, perturbed X plane at one pixel,
  perturbed R plane, one NaN Z, constant Z).
- [ ] Integration test: run the real `reconcile()` (idw) on a synthetic

### Task 7: quadtree plan (`tiles3d/quadtree.py`)

**Interfaces:**
- Produces:
  ```python
  @dataclass(frozen=True)
  class TilePlan:
      level: int; tx: int; ty: int          # ty/tx = row/col split index path
      col0: int; row0: int; col1: int; row1: int   # INCLUSIVE pixel span
      stride: int                            # 2**(leaf_depth - level)
      children: tuple[TilePlan, ...]         # () for leaves

  @dataclass(frozen=True)
  class TreePlan:
      root: TilePlan; levels: int; tile_count: int

  def plan_quadtree(width: int, height: int, tile_pixels: int = 256) -> TreePlan
  def sample_indices(start: int, stop_inclusive: int, stride: int) -> npt.NDArray[np.int64]
  def geometric_error(stride: int, pixel_size: float) -> float
  ```
- Rules: depth `L = smallest k ≥ 0 with (max(w,h) - 1) / 2**k <= tile_pixels`
  … concretely `L = 0; while max(width, height) > tile_pixels * 2**L: L += 1`.
  Root spans `[0, w-1]×[0, h-1]`. A tile at level < L splits each axis at
  `mid = lo + (hi - lo) // 2` into `[lo, mid]` and `[mid, hi]` (shared
  boundary index) — an axis with `hi == lo` doesn't split (2 or 4 children).
  `sample_indices` = `np.arange(start, stop_inclusive + 1, stride)` with
  `stop_inclusive` appended when not already the last element.
  `geometric_error(1, px) == 0.0`; else `stride * pixel_size * 4.0`.
- [ ] Tests: single-leaf grid (6×6); 600×600 → L=2, 16 leaves; leaf spans
  union-cover `[0,w-1]²` with shared-boundary-only overlap (parametrized
  over (6,6),(257,257),(600,300),(1,300)); strides halve per level;
  `sample_indices(0, 10, 4) == [0,4,8,10]`; GE monotone, leaves 0.
- [ ] `make check`; commit `feat(tiles3d): quadtree tiling plan`.

### Task 8: tile mesh (`tiles3d/mesh.py`)

**Interfaces:**
- Produces:
  ```python
  @dataclass(frozen=True)
  class TileMesh:
      positions: npt.NDArray[np.float32]   # (n, 3) glTF y-up, RTC-relative
      uvs: npt.NDArray[np.float32]         # (n, 2)
      indices: npt.NDArray[np.uint32]      # (t*3,)
      center: tuple[float, float, float]   # glTF-frame RTC node translation
      region: tuple[float, float, float, float, float, float]
      cols: npt.NDArray[np.int64]; rows: npt.NDArray[np.int64]  # sampled

  def build_tile_mesh(terrain: TerrainGrid, tile: TilePlan, geodesy: Geodesy) -> TileMesh
  ```
- Algorithm: `cols = sample_indices(tile.col0, tile.col1, tile.stride)`,
  same for rows; gather x/y/z at the sample grid (float64);
  `ex, ey, ez = geodesy.to_ecef(...)`; center_ecef = componentwise
  `(min+max)/2`; ECEF→glTF swizzle `g = (e_x, e_z, -e_y)` applied to BOTH
  the center (kept float64 → node translation) and the relative vertices
  (`(e - center)` in float64, swizzle, then `.astype(np.float32)`), so the
  runtime x-axis π/2 rotation reproduces ECEF exactly.
  UV for sample (j, i): `u = (i + 0.5) / len(cols)`, `v = (j + 0.5) / len(rows)`.
  Indices per cell (j, i), vertices row-major:
  `a = j*nc+i, b = a+1, c = a+nc, d = c+1` → triangles `(a, c, d), (a, d, b)`.
  Region from `to_geodetic_radians` over the same samples:
  `(lon.min(), lat.min(), lon.max(), lat.max(), h.min(), h.max())` reordered
  to `[west, south, east, north, minH, maxH]`.
- [ ] Tests: vertex count `len(cols)*len(rows)`; triangle count
  `2*(nc-1)*(nr-1)`; swizzle round-trip: rotate positions+center back by
  x-axis π/2 and compare to recomputed ECEF (≤ float32 eps · |center|);
  UVs in (0,1); region contains every vertex's geodetic coordinate;
  determinism.
- [ ] `make check`; commit `feat(tiles3d): stride-sampled RTC tile meshes`.

### Task 9: deterministic PNG codec (`tiles3d/png.py`)

**Interfaces:**
- Produces:
  ```python
  def encode_png(rgb: npt.NDArray[np.uint8]) -> bytes   # (h, w, 3)
  def decode_png(data: bytes) -> npt.NDArray[np.uint8]  # strict, Tiles3dError
  ```
- Encoder: signature `89 50 4E 47 0D 0A 1A 0A`; IHDR (w, h, bit depth 8,
  colour type 2, compression 0, filter 0, interlace 0); one IDAT holding
  `zlib.compress(scanlines, _ZLIB_LEVEL)` where scanlines = each row
  prefixed with filter byte 0; IEND. CRC-32 via `zlib.crc32` over
  chunk-type+data. `_ZLIB_LEVEL = 6` (pinned for determinism).
- Decoder (for our own files, strict): verify signature, walk chunks
  verifying length bounds and **every CRC**, require exactly
  IHDR→IDAT(s)→IEND, IHDR fields as above, inflate, verify decoded length
  `h*(1+3*w)`, every filter byte 0, return `(h, w, 3)` array. Distinct
  `Tiles3dError` per violation.
- [ ] Tests: round-trip bit-exact; bytes deterministic; negatives (bad
  signature, flipped CRC, wrong colour type, nonzero filter byte,
  truncated IDAT, trailing chunk after IEND, decoded-length mismatch).
- [ ] `make check`; commit `feat(tiles3d): deterministic PNG codec`.

### Task 10: glb assembly (`tiles3d/gltf.py`)

**Interfaces:**
- Produces:
  ```python
  def build_glb(mesh: TileMesh, png: bytes) -> bytes
  ```
- BIN layout, in order, each section padded to 4 bytes with `\x00`:
  positions (`n*12`), uvs (`n*8`), indices (`t*3*4`), png. bufferViews
  0..3 in that order (targets 34962, 34962, 34963, none). Accessors:
  0 POSITION (VEC3/5126, `min`/`max` = exact componentwise extremes of the
  float32 data as Python floats), 1 TEXCOORD_0 (VEC2/5126), 2 indices
  (SCALAR/5125). `images: [{bufferView: 3, mimeType: "image/png"}]`,
  `samplers: [{magFilter: 9729, minFilter: 9729, wrapS: 33071, wrapT: 33071}]`,
  `textures: [{sampler: 0, source: 0}]`,
  `materials: [{pbrMetallicRoughness: {baseColorTexture: {index: 0},
  metallicFactor: 0.0, roughnessFactor: 1.0}, doubleSided: true}]`,
  `meshes: [{primitives: [{attributes: {POSITION: 0, TEXCOORD_0: 1},
  indices: 2, material: 0, mode: 4}]}]`,
  `nodes: [{mesh: 0, translation: [cx, cy, cz]}]`, `scenes: [{nodes: [0]}]`,
  `scene: 0`, `asset: {generator: "ahn_cli tiles3d", version: "2.0"}`.
  JSON chunk: `json.dumps(..., sort_keys=True, separators=(",", ":"))`,
  space-padded to 4. GLB header magic `0x46546C67`, version 2, total
  length; JSON chunk type `0x4E4F534A`; BIN chunk type `0x004E4942`.
- [ ] Tests: header/length/alignment; parse JSON chunk back and check

### Task 12: orchestrator (`tiles3d/build.py`)

  @dataclass(frozen=True)

### Task 13: the strictest output verifier (`tiles3d/verify.py`) wired into build

**Files:**
- Create: `ahn_cli/tiles3d/verify.py`
- Modify: `ahn_cli/tiles3d/build.py` (call `verify_tiles3d(out, ortho,
  heights, tile_pixels=tile_pixels)` after `write_tileset`, inside the
  cleanup guard)
- Test: `tests/tiles3d/test_verify.py`

**Interfaces:**
- Produces: `def verify_tiles3d(out_dir: Path, ortho: Path, heights: Path,
  *, tile_pixels: int = 256) -> None` raising `Tiles3dError`.
- Checks (spec §verifier, all from FRESH disk reads +
  `load_terrain`/`plan_quadtree` recomputation):
  1. tileset.json: parses; **exact key sets** (`asset` exactly
     {generator, version}; tile entries exactly the keys `tile_entry`
     emits; unknown keys are errors); `asset.version == "1.1"`; root
     `refine == "REPLACE"`; every GE finite ≥ 0; child GE ≤ parent GE;
     tileset GE ≥ root GE.
  2. regions: `west < east`, `south < north`, lon ∈ [−π, π],
     lat ∈ [−π/2, π/2], `minH ≤ maxH`; child region ⊆ parent region.
  3. content uris resolve inside `out_dir` (reject `..`/absolute); the set
     of on-disk `*.glb` under `out_dir` equals the referenced set, each
     referenced exactly once.
  4. glb container: magic/version; declared length == file size; JSON then
     BIN chunk, 4-aligned, nothing after.
  5. glTF internals: buffer byteLength == BIN length; bufferViews within
     buffer; accessors within views; POSITION min/max **bit-exact** vs
     recomputed extremes of the payload; index count % 3 == 0; all indices
     < vertex count; every triangle has 3 distinct indices; UVs ∈ [0, 1].
  6. PNG: full strict `decode_png` (CRCs!), dims == sampled tile dims,
     pixels bit-exact vs the freshly sampled ortho.
  7. geometry ground truth: `build_tile_mesh` recomputed → positions, uvs,
     indices, node translation bit-exact vs the glb payload.
  8. containment: every vertex's geodetic coordinate (recomputed) inside
     the tile's stored region AND every ancestor's stored region.
  9. full coverage: recomputed leaf spans tile `[0,w-1]×[0,h-1]` exactly
     (assembled boolean grid: every pixel covered).
