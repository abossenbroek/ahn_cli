# Rust-consumer tiles3d fixtures

Tiny, committed `game` and `heightfield` tilesets for the **separate Rust
runtime repo** to use as its own decode fixtures. Rust-side decoding is out
of this repo's CI scope; these files exist so that repo has a known-good,
byte-stable input to test its `.glb` / `.hf` / `.jpg` readers against.

## What's here

- `game/` — a `--profile game` tileset: `tileset.json`, five
  `KHR_mesh_quantization` + `EXT_meshopt_compression` glb tiles with
  embedded baseline JPEG textures, and the deterministic `provenance.json`.
- `heightfield/` — a `--profile heightfield` tileset: `tileset.json`, five
  `.hf` height chunks (see
  `docs/superpowers/specs/2026-07-12-heightfield-chunk-format.md` — the
  normative byte layout the Rust decoder codes against), five sibling
  baseline `.jpg` textures, and `provenance.json`.

Both are the byte-freeze scene: a 12×12 synthetic ortho/EXR pair at
`tile_pixels=8`, which yields a two-level quadtree (one root + four leaves).

## Provenance and the pinned-geodesy caveat

These fixtures are generated by `build_tiles3d` through the real pipeline,
but with geodesy **pinned** to a deterministic pure-numpy spherical-Earth
model (no PROJ). Real geodesy is only self-consistent *per machine* (the
NLGEO2018 quasi-geoid grid may or may not be installed — see
`ahn_cli/tiles3d/geodesy.py`), so real-geodesy output could not be committed
and byte-checked across machines. The pin lives in
`tests/tiles3d/test_integration_profiles.py` (`pin_geodesy` and the
`_fake_to_*` transforms) and is a genuine spherical model — all three
transforms are exact inverses — so the game profile's quantization-bound
containment verifier passes. **Absolute ECEF/geodetic coordinates in these
fixtures are therefore synthetic**, not real Dutch positions; only the
container framing, codec streams, and format layout are representative.

## Regeneration

`tests/tiles3d/test_integration_profiles.py::test_committed_fixtures_are_not_stale`
rebuilds these with the same pin and byte-compares them on every `make
test` run, so unintended drift (a codec or format change) is a red test.
When a change is intentional, regenerate in one shot:

```bash
uv run python -m tests.tiles3d.regen_rust_fixtures
```
